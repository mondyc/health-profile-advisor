# Health Profile Advisor 🏥📊

> 交互式健康画像构建与体检报告分析技能 — 智能追问收集用药/就医/检查/家族病史，解析单次或多年体检报告趋势，生成个性化健康建议与体检方案

## 功能概览

### 🗂️ 健康画像构建（Phase 1）

采用**智能追问式表单**交互，而非固定问题列表：

- **用药记录** 💊 — 有则追问药名、频次、效果，根据药物类别关联追问（降压药→血压控制，他汀→血脂情况）
- **就医记录** 🏥 — 逐条追问原因→科室→诊断→后续，持续收集直到用户说停
- **检查检测记录** 🔬 — 专项检查详情，异常结果后续与体检报告对比
- **家族病史** 👨‍👩‍👦 — 直系亲属慢性病、肿瘤史，关联风险评估
- **生活习惯** 🚬 — 吸烟、饮酒、运动，仅对有风险的习惯追问细节

每个模块提供选项降低回答负担，同时允许自由输入，根据用户的每次回答智能生成下一步问题。

### 📋 体检报告解析（Phase 2）

- 读取 PDF/图片/Word 格式体检报告
- 提取所有检验指标，标记异常项
- 结合健康画像判断临床意义（如：空腹血糖偏高 + 糖尿病家族史 → 重点关注）
- 影像报告提取检查部位、检查所见、诊断意见

### 📈 跨年趋势分析（Phase 3）

- 同一指标按年份排列，识别趋势类型：持续上升 📈 / 持续下降 📉 / 波动 📊 / 稳定 ➡️ / 突变 ⚡
- 风险信号识别：趋势指向异常阈值、正常转异常、多指标联动变化
- 关联分析：如"血脂↑与 BMI↑同步，提示代谢综合征风险"

### 💡 健康建议生成（Phase 4）

- **生活注意事项**：饮食、运动、作息，每条说明关联指标
- **体检项目建议**：必查项目（异常复查）+ 加查项目（基于画像风险）+ 常规项目
- **就医建议**：仅在指标严重偏离、趋势恶化等情况下给出
- **下次体检提醒**：时间 + 重点复查 + 加查项目

### SKILL.md

技能核心定义文件，包含：
- YAML frontmatter（触发条件与适用边界）
- Phase 1~4 完整流程与输出格式
- 安全与边界规则（不做诊断、不推荐药物、不处理急症等）
- 工作流示例

### references/interview-guide.md

Phase 1 五个交互模块的详细追问逻辑：
- 每个模块的起始问题（含选项）
- 追问逻辑表（根据用户回答动态路由）
- 追问原则与示例

### references/medical-indicators.md

8 大类检验指标参考范围：
- 血常规、肝功能、肾功能、血脂、血糖代谢、肿瘤标志物、尿常规、其他（甲状腺/心肌/电解质）
- 每类包含解读要点

### references/exam-recommendations.md

分年龄段体检项目推荐：
- 基础必查项目（所有成年人）
- 分年龄段加查（20-29 / 30-39 / 40-49 / 50-59 / 60+）
- 家族史加查项目
- 生活方式风险加查项目
- 复查时间建议

## 安全边界

| 规则 | 说明 |
|------|------|
| ❌ 不做诊断 | 使用"提示"、"可能"措辞，绝不下确定性诊断 |
| ❌ 不推荐药物 | 用药问题一律建议咨询医生 |
| ❌ 不处理急症 | 急性症状建议拨打急救电话 |
| ✅ 肿瘤标志物慎重 | 轻度升高≠癌症，必须注明免责 |
| ✅ 每份报告附免责声明 | "仅供参考，不能替代专业医疗诊断" |

## 技术说明

本技能为 [ima.copilot](https://ima.copilot) 平台的 Skill 格式，遵循以下设计原则：

- **渐进式披露**：SKILL.md 始终加载，references 按需加载
- **SKILL.md ≤ 400 行**：详细内容拆入 references
- **流程型结构**：四个阶段按需执行，支持多种进入时机
- **确认门设计**：画像汇总需用户确认后再进入分析

## 适用场景

- 个人健康管理与年度体检规划
- 家庭成员健康档案建立
- 多年体检数据纵向对比分析
- 代谢综合征早期信号识别与干预建议

## 不适用场景

- 疾病诊断与处方开具
- 紧急医疗咨询
- 单一指标百科式问答（应引导进入完整流程）
